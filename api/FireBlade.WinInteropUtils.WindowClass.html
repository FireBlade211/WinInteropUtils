Remarks

Note that the Atom property of the created WindowClass may be 0. This is not an error. Windows assigns an atom only when a class is registered through Register(). If the class name already exists — such as a built-in window class or one registered internally by Windows (e.g. through InitCommonControls) — the system does not return an atom, and Windows doesn't provide an API to query it afterward.

Because Windows does not expose class atoms for pre-existing classes, the library cannot provide an atom value for them and Atom is set to 0.
