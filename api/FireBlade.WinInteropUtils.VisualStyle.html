DrawText(Graphics, string, ref Rectangle, ThemeTextOptions)

Draws formatted text in the specified rectangle without visual styles. Formats the text according to the specified method (expanding tabs, justifying characters, breaking lines, and so forth).

public static int DrawText(Graphics gh, string text, ref Rectangle rect, ThemeTextOptions options)

Parameters

gh Graphics

The Graphics surface to draw on.

text string

A string that specifies the text to be drawn.

rect Rectangle

A Rectangle structure that contains the rectangle (in logical coordinates) in which the text is to be formatted.

options ThemeTextOptions

The method of formatting the text.

Returns

int

If the function succeeds, the return value is the height of the text in logical units. If AlignMiddle or AlignBottom is specified, the return value is the offset from lpRect->top to the bottom of the drawn text. If the function fails, the return value is zero.

Remarks

Unless the NoClip option is used, DrawText clips the text so that it does not appear outside the specified rectangle. Note that text with significant overhang may be clipped, for example, an initial "W" in the text string or text that is in italics. All formatting is assumed to have multiple lines unless the SingleLine option is specified.

If the selected font is too large for the specified rectangle, the function does not attempt to substitute a smaller font.

OpenThemeData(Window, string)

Opens the theme data for a window and its associated class.

public static VisualStyle OpenThemeData(Window wnd, string classList)

Parameters

wnd Window

The window for which theme data is requested.

classList string

A semicolon-seperated list of classes. See the Visual Styles article for more info.

Returns

VisualStyle

The visual style opened.

Remarks

OpenThemeData(Window, string) tries to match each class, one at a time, to a class data section in the active theme. If a match is found, an associated VisualStyle is returned. If no match is found, a FileNotFoundException is thrown. You should guard against this in a try-catch block, and fall back to the static non-visual-style dependent functions in the catch block instead, such as DrawText(Graphics, string, ref Rectangle, ThemeTextOptions).

The classList parameter contains a list, not just a single name, to provide the class an opportunity to get the best match between the class and the current visual style. For example, a button may pass "OkButton;Button" if its ID is ID_OK. If the current visual style has an entry for OkButton, that is used; otherwise no visual style is applied.

Exceptions

FileNotFoundException

A matching class data section was not found.
